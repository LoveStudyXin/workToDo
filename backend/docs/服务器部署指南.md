## 1. 服务器准备

### 1.1 连接服务器

```bash
ssh root@你的服务器IP
# 或使用密钥
ssh -i ~/.ssh/your_key root@你的服务器IP
```

### 1.2 安装基础软件

```bash
# 更新系统
apt update && apt upgrade -y

# 安装必要工具
apt install -y git curl wget vim

# 安装 Nginx
apt install -y nginx

# 安装 Python 3.11+
apt install -y python3 python3-pip python3-venv

# 安装 Node.js (用于构建前端)
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs
```

### 1.3 创建项目目录

```bash
mkdir -p /var/www/work-todo
cd /var/www/work-todo
```

### 1.4 上传代码

**方式一：Git 克隆**

```bash
git clone https://github.com/你的用户名/work-todo.git .
```

**方式二：本地上传**

```bash
# 在本地执行
scp -r ./backend root@服务器IP:/var/www/work-todo/
scp -r ./frontend root@服务器IP:/var/www/work-todo/
```

## 2. 后端部署

### 2.1 创建虚拟环境

```bash
cd /var/www/work-todo/backend
python3 -m venv venv
source venv/bin/activate
```

### 2.2 安装依赖

```bash
pip install -r requirements.txt
```

### 2.3 配置环境变量

创建 `.env` 文件：

```bash
vim .env
```

内容：

```env
SECRET_KEY=你的随机密钥字符串
DATABASE_URL=sqlite+aiosqlite:///./work_todo.db
ACCESS_TOKEN_EXPIRE_MINUTES=10080
```

生成随机密钥：

```bash
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```

### 2.4 初始化数据库

```bash
# 应用数据库迁移
python -m alembic upgrade head
```

### 2.5 测试运行

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

访问 `http://服务器IP:8000/docs` 确认 API 正常。

### 2.6 配置 Systemd 服务

创建服务文件：

```bash
vim /etc/systemd/system/work-todo-api.service
```

内容：

```ini
[Unit]
Description=Work Todo API
After=network.target

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/var/www/work-todo/backend
Environment="PATH=/var/www/work-todo/backend/venv/bin"
ExecStart=/var/www/work-todo/backend/venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 修改目录权限
chown -R www-data:www-data /var/www/work-todo/backend

# 重载 systemd
systemctl daemon-reload

# 启动服务
systemctl start work-todo-api

# 开机自启
systemctl enable work-todo-api

# 查看状态
systemctl status work-todo-api

# 查看日志
journalctl -u work-todo-api -f
```

## 3. 前端部署

### 3.1 修改 API 地址

编辑前端配置，将 API 地址改为生产环境：

```bash
vim /var/www/work-todo/frontend/.env.production
```

内容：

```env
VITE_API_BASE_URL=https://你的域名
```

或直接修改 `src/utils/api.ts` 中的 baseURL。

### 3.2 构建前端

```bash
cd /var/www/work-todo/frontend
npm install
npm run build
```

构建产物在 `dist` 目录。

### 3.3 部署静态文件

```bash
# 复制到 Nginx 目录
cp -r dist /var/www/work-todo/frontend-dist

# 修改权限
chown -R www-data:www-data /var/www/work-todo/frontend-dist
```

## 4. Nginx 配置

### 4.1 创建站点配置

```bash
vim /etc/nginx/sites-available/work-todo
```

内容：

```nginx
server {
    listen 80;
    server_name 你的域名;  # 例如: todo.example.com

    # 前端静态文件
    root /var/www/work-todo/frontend-dist;
    index index.html;

    # 前端路由 (Vue Router history 模式)
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持 (如需要)
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

### 4.2 启用站点

```bash
# 创建软链接
ln -s /etc/nginx/sites-available/work-todo /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重载 Nginx
systemctl reload nginx
```

## 5. 域名配置

### 5.1 购买域名

常用域名注册商：
- 阿里云：https://wanwang.aliyun.com/
- 腾讯云：https://dnspod.cloud.tencent.com/
- Cloudflare：https://www.cloudflare.com/

### 5.2 DNS 解析

在域名管理后台添加 DNS 记录：

| 记录类型 | 主机记录 | 记录值 | TTL |
|---------|---------|--------|-----|
| A | @ 或 todo | 你的服务器IP | 600 |

例如：
- 主域名：`example.com` → 主机记录填 `@`
- 子域名：`todo.example.com` → 主机记录填 `todo`

### 5.3 验证解析

```bash
# 等待 DNS 生效后 (通常几分钟到几小时)
ping 你的域名
nslookup 你的域名
```

## 6. SSL 证书

### 6.1 安装 Certbot

```bash
apt install -y certbot python3-certbot-nginx
```

### 6.2 申请证书

```bash
certbot --nginx -d 你的域名
```

按提示操作：
1. 输入邮箱
2. 同意服务条款
3. 选择是否重定向 HTTP 到 HTTPS（建议选是）

### 6.3 自动续期

Certbot 会自动配置续期任务，可手动测试：

```bash
certbot renew --dry-run
```

### 6.4 最终 Nginx 配置

Certbot 会自动修改配置，最终类似：

```nginx
server {
    listen 80;
    server_name 你的域名;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name 你的域名;

    ssl_certificate /etc/letsencrypt/live/你的域名/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/你的域名/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    root /var/www/work-todo/frontend-dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

## 7. 常用运维命令

### 服务管理

```bash
# 后端服务
systemctl start work-todo-api    # 启动
systemctl stop work-todo-api     # 停止
systemctl restart work-todo-api  # 重启
systemctl status work-todo-api   # 状态
journalctl -u work-todo-api -f   # 实时日志

# Nginx
systemctl reload nginx           # 重载配置
systemctl restart nginx          # 重启
nginx -t                         # 测试配置
```

### 更新部署

```bash
# 更新后端
cd /var/www/work-todo/backend
git pull  # 或重新上传文件
source venv/bin/activate
pip install -r requirements.txt
python -m alembic upgrade head
systemctl restart work-todo-api

# 更新前端
cd /var/www/work-todo/frontend
git pull  # 或重新上传文件
npm install
npm run build
rm -rf /var/www/work-todo/frontend-dist/*
cp -r dist/* /var/www/work-todo/frontend-dist/
```

### 数据库备份

```bash
# 备份
cp /var/www/work-todo/backend/work_todo.db ~/backups/work_todo_$(date +%Y%m%d).db

# 定时备份 (crontab)
crontab -e
# 添加: 0 2 * * * cp /var/www/work-todo/backend/work_todo.db /root/backups/work_todo_$(date +\%Y\%m\%d).db
```

### 查看日志

```bash
# 后端日志
journalctl -u work-todo-api --since "1 hour ago"

# Nginx 访问日志
tail -f /var/log/nginx/access.log

# Nginx 错误日志
tail -f /var/log/nginx/error.log
```

## 快速部署脚本

创建 `deploy.sh`：

```bash
#!/bin/bash
set -e

echo "=== 部署开始 ==="

# 后端
echo ">>> 更新后端..."
cd /var/www/work-todo/backend
source venv/bin/activate
pip install -r requirements.txt -q
python -m alembic upgrade head
systemctl restart work-todo-api

# 前端
echo ">>> 构建前端..."
cd /var/www/work-todo/frontend
npm install --silent
npm run build
rm -rf /var/www/work-todo/frontend-dist/*
cp -r dist/* /var/www/work-todo/frontend-dist/

echo "=== 部署完成 ==="
```

使用：

```bash
chmod +x deploy.sh
./deploy.sh
```

## 常见问题

### Q: 502 Bad Gateway

**原因**：后端服务未运行或端口不对

**解决**：
```bash
systemctl status work-todo-api
journalctl -u work-todo-api -n 50
```

### Q: 前端页面刷新 404

**原因**：Nginx 未配置 Vue Router history 模式

**解决**：确保 `location /` 中有 `try_files $uri $uri/ /index.html;`

### Q: API 跨域错误

**原因**：后端 CORS 配置问题

**解决**：检查 FastAPI 的 CORS 中间件配置，确保允许你的域名

### Q: SSL 证书过期

**解决**：
```bash
certbot renew
systemctl reload nginx
```

### Q: 磁盘空间不足

```bash
# 查看磁盘使用
df -h

# 清理日志
journalctl --vacuum-time=7d

# 清理 apt 缓存
apt clean
```
